Transparência Social

Fluxo completo do sistema — jornada dos perfis, regras de negócio e fluxo de aprovação de relatórios.

Versão 1.0  •  Abril 2026  •  Material para alinhamento com stakeholders
SEÇÃO 1

Resumo Executivo

O Transparência Social é uma plataforma de prestação de contas para projetos sociais. A Organização responsável pelo projeto cria relatórios periódicos, preenche os dados de execução e financeiros, e os envia para análise.

A análise e aprovação segue uma regra simples: se o projeto possui um Consultor vinculado, é o Consultor quem analisa e aprova. Se não há Consultor, o Investidor assume esse papel diretamente.

Caso o relatório seja devolvido para ajustes, a Organização corrige e reenvia quantas vezes for necessário, até a aprovação final. Todos os dados financeiros preenchidos nos relatórios são automaticamente consolidados na aba Financeiro do projeto, oferecendo uma visão completa e atualizada dos recursos aplicados.

SEÇÃO 2

Fluxo Geral do Sistema

O ciclo de vida de um relatório no Transparência Social segue uma jornada bem definida, com quatro status possíveis e papéis claros em cada etapa.

Status do Relatório

Rascunho → Enviado → Aprovado
Enviado → Devolvido → Rascunho (ciclo de ajustes)

Jornada Completa

A Organização cria um novo relatório para o projeto, que nasce com status Rascunho. Neste estágio, a Organização preenche todas as seções: dados gerais de execução, itens financeiros (orçamento planejado, gastos, remanejamentos), comprovantes de despesas (notas e recibos) e extratos bancários.

Ao concluir o preenchimento, a Organização envia o relatório para análise. O status muda para Enviado, e o relatório fica bloqueado para edição. Neste ponto, o sistema verifica se o projeto possui um Consultor ativo vinculado.

Se houver Consultor, ele é quem receberá o relatório para análise. Caso contrário, o Investidor assume a análise. O analista responsável tem duas opções: aprovar o relatório (status final: Aprovado) ou devolver para ajustes (status: Devolvido), incluindo um comentário explicando o que precisa ser corrigido.

Se devolvido, a Organização recebe o relatório de volta, faz os ajustes necessários e reenvia. Esse ciclo se repete até a aprovação final. Ao ser aprovado, o relatório é travado e todos os dados financeiros ficam disponíveis na visão consolidada do projeto.

SEÇÃO 3

Fluxo por Perfil

Perfil

Organização

A Organização é o perfil operacional do sistema. É sempre ela quem inicia e conduz o preenchimento dos relatórios.

  • Cria novos relatórios dentro dos projetos que administra
  • Preenche todas as seções do relatório: dados de execução, itens financeiros, comprovantes e extratos
  • Envia o relatório para análise quando o preenchimento está completo
  • Recebe de volta relatórios devolvidos, corrige o que for necessário e reenvia
  • Não participa da decisão de aprovação — essa responsabilidade é do Consultor ou do Investidor

Quando o relatório é devolvido: A Organização recebe o relatório com status "Devolvido" e o comentário do analista. Pode então editar o conteúdo, corrigir os pontos indicados e reenviar para nova análise. O ciclo se repete quantas vezes for necessário.

Perfil

Consultor

O Consultor é um perfil especializado, vinculado a projetos específicos. Nem todo projeto terá um Consultor.

  • Entra no fluxo apenas quando está vinculado e ativo em um projeto específico
  • Analisa os relatórios enviados pela Organização
  • Aprova ou devolve os relatórios, com comentário obrigatório ao devolver
  • Substitui o Investidor na função de análise e aprovação — quando existe Consultor no projeto, o Investidor não participa da aprovação
  • Não cria, não edita, não preenche relatórios — seu papel é exclusivamente de análise

Cenário de atuação: O Consultor atua como uma camada de verificação especializada. Quando presente no projeto, ele assume integralmente a responsabilidade de aprovação, liberando o Investidor dessa tarefa operacional.

Perfil

Investidor

O Investidor é o financiador do projeto. Seu nível de envolvimento na aprovação depende da existência de um Consultor.

  • Analisa e aprova diretamente quando o projeto não possui Consultor vinculado
  • Não participa da aprovação quando o projeto possui Consultor — nesse caso, o Consultor é o responsável
  • Tem visibilidade total sobre os projetos, relatórios e dados financeiros consolidados
  • Pode criar relatórios e editar rascunhos, quando necessário

Regra principal: O Investidor atua diretamente na aprovação apenas nos projetos sem Consultor. Nos projetos com Consultor, ele mantém visibilidade completa, mas a decisão de aprovação é delegada ao Consultor.

SEÇÃO 6

Regras de Negócio Consolidadas

1
A Organização é sempre quem cria e preenche o relatório. Nenhum outro perfil inicia o preenchimento operacional.
2
O relatório nasce com status Rascunho e pode ser editado livremente pela Organização até o envio.
3
Para enviar um relatório, é obrigatório que exista pelo menos um comprovante de despesa (nota fiscal ou recibo) anexado.
4
Se o projeto possui Consultor ativo vinculado, é o Consultor quem analisa e toma a decisão final de aprovar ou devolver o relatório.
5
Se o projeto não possui Consultor, o Investidor assume diretamente a análise e aprovação.
6
Apenas um analista por vez é responsável pela aprovação: ou o Consultor, ou o Investidor — nunca ambos simultaneamente.
7
Ao devolver um relatório, o analista deve incluir um comentário explicando os ajustes necessários.
8
Relatórios devolvidos voltam para a Organização com status Devolvido. A Organização pode editar e reenviar quantas vezes for necessário.
9
Os dados financeiros são preenchidos dentro dos relatórios, não diretamente na aba Financeiro do projeto.
10
A aba Financeiro do projeto exibe uma visão consolidada automaticamente, agregando os dados de todos os relatórios do projeto.
11
Nem todo projeto terá Consultor. A presença ou ausência do Consultor define quem é o responsável pela aprovação.
12
O relatório aprovado fica travado para edição. Nenhum perfil pode alterar um relatório já aprovado.
13
O sistema mantém versionamento completo de cada relatório, registrando todas as versões desde o rascunho inicial até a aprovação.
SEÇÃO 7

Versão Resumida para Apresentação

Bruno,

Segue o resumo do fluxo final do Transparência Social, conforme definido e implementado.

O sistema funciona assim: a Organização cria o relatório do projeto, preenche todos os dados — incluindo a parte financeira (itens de orçamento, gastos, comprovantes e extratos) — e envia para análise.

A análise funciona de forma dinâmica, dependendo da configuração do projeto. Se existe um Consultor vinculado ao projeto, é ele quem analisa e decide: aprova ou devolve. Se não existe Consultor, o Investidor assume esse papel e faz a aprovação diretamente.

Se o relatório for devolvido, a Organização recebe de volta com os comentários do analista, faz os ajustes e reenvia. Esse ciclo se repete até a aprovação.

Uma vez aprovado, o relatório fica travado e todos os dados financeiros são consolidados automaticamente na aba Financeiro do projeto — dando visibilidade total sobre a aplicação dos recursos.

As permissões estão bem definidas: a Organização é operacional (cria, preenche, envia), e a aprovação é sempre de um único responsável por projeto (Consultor quando presente, Investidor quando não).
